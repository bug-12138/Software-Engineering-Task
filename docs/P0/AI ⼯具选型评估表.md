
#  AI 工具选型评估表

根据项目启动阶段的任务要求，我们对四类 AI 工具进行了横向评估。以下是我们的最终选型建议及理由：

| 工具类型 | 候选工具 | 是否选用 | 评估与选用/弃用理由 (基于功能、易用性、成本、团队熟悉度) |
| :--- | :--- | :---: | :--- |
| **AI 编程助手** | **GitHub Copilot** | **✅ 选用** | **选用理由：** 团队对 GitHub 生态非常熟悉 。它在 IDEA、CLion 等常用 IDE 中集成度极高，代码补全延迟低，非常适合 P4/P5 阶段核心业务逻辑的快速编码 。 |
| | Cursor | ❌ 弃用 | **弃用理由：** 作为一个独立的编辑器，要求全组同学放弃原本习惯的本地 IDE 配置进行整体迁移，前期磨合与环境配置成本偏高，不符合 10 周短周期的进度要求 。 |
| **AI 对话工具** | **Gemini** | **✅ 选用** | **选用理由：** 具备极强的长文本上下文处理能力。在 P1 阶段处理长篇幅的《需求规格说明书》或在开发期分析复杂的 Docker 报错日志时，表现更为稳定且不易遗忘前置设定 。 |
| | ChatGPT (免费版) | ❌ 弃用 | **弃用理由：** 免费版本在高峰期响应偶有延迟，且上下文窗口存在限制，在生成涉及 15 条以上用户故事或复杂系统类图描述时，可能出现输出截断或逻辑断层 。 |
| **AI 代码审查工具** | **CodeRabbit** | **✅ 选用** | **选用理由：** 易用性佳，可直接作为 App 接入 GitHub 仓库。它能在每次 PR 时自动进行逐行级审查，有效拦截潜在的异常，大幅减轻开发负责人在 P4 阶段的审查压力。 |
| | 团队自建 CI 调用 API | ❌ 弃用 | **弃用理由：** 虽然配置灵活，但自行编写脚本和处理 API 额度/网络问题会额外耗费不少工时，考虑到每周人均仅 6 小时的工时限制，应优先选择成熟的集成方案 。 |
| **AI 测试工具** | **CodiumAI** | **✅ 选用** | **选用理由：** 功能专注，作为 IDE 插件可直接生成详尽的单元测试，并自动考虑边界条件，非常符合项目对代码质量保障和测试覆盖率的要求。 |
| | GitHub Copilot Chat | ❌ 弃用 | **弃用理由：** 尽管已选用 Copilot 辅助编码，但其对话生成的测试代码在自动分析复杂执行流和补充异常分支方面略显笼统，不如专门针对测试场景优化的工具精准 。 |
